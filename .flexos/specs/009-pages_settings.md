| Field | Type | Validation | Required |
|-------|------|------------|----------|
| Avatar | Image | JPG/PNG/GIF, max 2MB | No |
| Full Name | Text | 1-100 characters | Yes |
| Email | Email | Valid email, unique | Yes |

### Actions

| Action | Behavior |
|--------|----------|
| Upload Photo | Open file picker, upload to storage, update profile |
| Remove Photo | Clear avatar_url, show initials |
| Save Changes | Validate and save name/email changes |

### Email Change Flow

1. User enters new email
2. Click "Save Changes"
3. Verification email sent to new address
4. User clicks verification link
5. Email updated in profile

2. Confirmation modal appears

4. Account and all data deleted
5. User logged out, redirected to landing

## Data Requirements

```typescript
interface SettingsData {
  profile: {
    id: string
    name: string
    email: string
    avatar_url: string | null
  }
  preferences: {
    in_app_task_reminders: boolean
    in_app_task_assignments: boolean
    in_app_team_activity: boolean
    in_app_task_completions: boolean
    push_enabled: boolean
    email_weekly_digest: boolean
  }
}
```

## Validation

### Profile

| Field | Validation | Error Message |
|-------|------------|---------------|
| Name | Required, 1-100 chars | "Please enter your name" |
| Email | Valid format | "Please enter a valid email" |
| Email | Unique | "This email is already in use" |

### Password

| Field | Validation | Error Message |
|-------|------------|---------------|
| Current | Correct | "Current password is incorrect" |
| New | Min 8 chars | "Password must be at least 8 characters" |
| New | Has letter | "Password must contain a letter" |
| New | Has number | "Password must contain a number" |
| Confirm | Matches | "Passwords don't match" |

## Acceptance Criteria

### Profile
- [ ] User can view their current profile info
- [ ] User can upload/change avatar
- [ ] User can remove avatar
- [ ] User can change their name
- [ ] User can change their email (with verification)
- [ ] Changes are saved and persisted

### Notifications
- [ ] All toggle preferences are displayed
- [ ] Toggles save immediately
- [ ] Push notification toggle requests permission
- [ ] Preferences persist after page reload

### Account
- [ ] User can change password with current password verification
- [ ] Password strength indicator works
- [ ] User can delete account with confirmation
- [ ] Deleted user is logged out
